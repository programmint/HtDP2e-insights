## 2、Modulo 函数的内涵

(modulo a n) 的意思是 ： a / n 的模多少？

注意，取模运算和余数运算，二者并不一致。

**核心公式： a = q * n + r**

其中
* a 是被除数（dividend）
* n 是除数（divisor）
* q 是商（quotient）
* r 是模（result），即 modulo 的计算结果

**关键规则 1**：
* r 的符号，永远与除数 n 的符号相同。

**关键规则 2**：
  * 当 n 为正数时，r的值，永远位于 [0, n) 范围内。
  * 当 n 为负数时，r的值，永远位于 (n, 0] 范围内。
  * 无论被除数 a 有多大，r 都不会超出上述范围。

<br>

## 3、取模运算的4种情况

### 3.1、大正数 / 小正数

**（modulo 5 3）= 2** 

公式：a = q * n + r

规则: r （模）的符号，与 n （除数）的符号，必须相同

* 推导: 5 / 3 = 1.6666 (6 循环不断) ，q 是正数，向下取整为 1 
* 验证： 5 = 1 * 3 + 2，且 2 在 [0, 3) 内

* 结果:取模的值是 2

<br>

### 3.2、小正数 / 大正数

**（modulo 3 5） = 3** 

公式：a = q * n + r

规则: r （模）的符号，与 n （除数）的符号，必须相同

* 推导: 3 / 5 = 0.6 ，向下取整， q = 0 
* 验证：3 = 0 * 5 + 3，且 3 在 [0, 5) 内
* 结果：取模值就是它自己 3

<br>

### 3.3、负数 / 正数

**（modulo （-5）3） = 1**

公式：a = q * n + r

规则: r （模）的符号，与 n （除数）的符号，必须相同

* 推导：（-5） / 3 = -1.66 ，向下取整，q = -2 
* 验证：（-5） = （-2） * 3 + 1 ，且 1 在 [0, 3) 内
* 结果：1

<br>

### 3.4、正数 / 负数

**（modulo 3 （-5）） = -2**

公式：a = q * n + r

规则: r （模）的符号，与 n （除数）的符号，必须相同

* 推导：3 / （-5） = -0.6 ，向下取整，q = -1 
* 验证：3 = （-1） * （-5）+ (-2)  ，且 -2 在 (-5, 0] 内
* 结果：-2

<br>

## 4、何为向下取整

纵观上面4种情况，无论是除尽了，还是除不尽，都有一个向下取整的过程，所谓的**向下取整**，就是根据计算出来的商，**向数轴的左边走，找到第一个整数。**

例如：

情况 1：5 / 3 = 1.6666 (6 循环不断) ，根据 1.6666 这个值，向数轴左边走，最靠近的整数是 1，所以 q = 1

情况 3：（-5） / 3 = -1.66 ，根据 -1.66 这个值，还是向数轴的左边走，最靠近的整数是 -2，所以 q = -2

## 5、若不用 modulo ？ 

假设在 [0, WIDTH) 区间内循环移动。

### 5.1、cond 语句

```
;; 移动 x，如果超出 WIDTH，回到 0
;; Number -> Number
(define (move-right x speed)
  (cond
    [(> (+ x speed) WIDTH) 0]  ; 超出右边？回零。
    [else (+ x speed)]))       ; 没超？继续走。
```

**第1、** 考虑移动方向

**第2、** 考虑 speed 的值，x 的值， WIDTH 的值

**第3、** 每移动一次，就好核对一次有没有超出 WIDTH 的值，没有超出，则执行 else 语句。


### 5.2、modulo 函数

(modulo (+ speed x) WIDTH)

**首先、** 写法上就简单了很多，一行而已。
**其次、** 也考虑移动方向。
**再次、** 对于speed 、WIDTH 值大小，并不敏感 

综合对比，采用 modulo 函数，思维模式简单了很多。

### 5.3、遗漏条件

上面 cond 语句还遗漏了一条件，即默认了 speed < WIDTH，可假如 speed > WIDTH 呢？

如果 speed = 200，WIDTH = 100 ，cond 只是简单地把位置重置为 0。

但按物理逻辑，应该绕了两圈回到原点。Modulo 自动处理了这种绕圈，Cond 做不到。

### 5.4、对比总结

cond 语句，要考虑：

**第1、移动方向**
**第2、speed 值**
**第3、WIDTH值**
**第4、时时刻刻对比，新的 X 值 与 WIDTH值大小**

反观 modulo 函数：

**第1、不考量移动方向：** (modulo (+ x speed) WIDTH) 这行代码，不管 speed 是正还是负，也即不管向左移动，还是向右移动，写法一模一样！

**第2、卡 X 与 WIDTH 的比值范围：** 对 X 与 WIDTH 谁大谁小，modulo 不敏感，它只需要把比值卡在一个范围内即可。

**总结：**

* cond 至少顾虑 4 重因素

* modulo 函数只需考虑 1 层因素。

显而易见，**cond 语句复杂，modulo 则“轻舟已过万重山”。**

<br>

## 6、modulo 的数学原理

用 Racket 语言写下 (modulo …) 时，应感谢一个人，**数学王子高斯 (Carl Friedrich Gauss)**。

1801 年高斯的传世名作 **《算术研究》(Disquisitiones Arithmeticae)** 正式出版，其中系统性地介绍了取模运算。

请 AI 解释了其背后的完整数学理论，看懂的部分不多，可高斯一核心思路看清了，正是**转化**。

第 5 节中，cond 的思路是：
> "值超了没有？超了怎么办？超了多少？"
> ——数轴上一不断变化的点，cond 语句需要不断追踪。

而 modulo 的思路是：
> "不管值多大，直接告诉我它在循环中的位置。"
> ——不断增长的数，变成固定范围内的循环。

尽管看不懂背后的内涵，可从无限到有限，就是转化。
**换一个角度，复杂的事情突然变简单了，数学的力量。**
